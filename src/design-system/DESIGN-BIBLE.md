# Aesthetix Design Bible

> **Competence through restraint.**
>
> Every page should feel like it belongs to the same brand.
> These principles guide every decision.

---

## 01 — Philosophy

### What should every page make someone feel?

**Competence.**

Not beauty. Not excitement. Not trust.

*Competence.*

The feeling that these people know exactly what they're doing.

How do you create that feeling?

Through **restraint**.

Every element you add reduces competence.
Every element you remove increases it.

The best Aesthetix page is the one with the fewest things in it.

### The One-Sentence Test

If you can't describe what a page does in one sentence, it's doing too much.

**Wrong:**
> "This page shows our services, process, testimonials, pricing, team, and FAQ."

**Right:**
> "This page shows we build products that people remember."

### What we're NOT

| We are not | We are |
|------------|--------|
| An agency | A studio |
| A service provider | A partner |
| A vendor | A craftsperson |
| A portfolio | A publication |

The homepage should feel like opening a magazine, not browsing a catalog.

## 02 — Experience Principles

### How the website should think

Every interaction follows psychological principles, not design trends.

### The Five Principles

| Principle | Meaning | Application |
|-----------|---------|-------------|
| **Reduce friction** | Remove steps between desire and action | One CTA per section, not five |
| **Increase confidence** | Every element should make the visitor feel smarter | Specific numbers, not vague claims |
| **Reward curiosity** | Give more as the visitor gives attention | Progressive disclosure, not everything at once |
| **Guide attention** | Control where the eye goes next | White space, size, contrast |
| **Create momentum** | Make the next action feel inevitable | Narrative flow, not random sections |

### The Anti-Patterns

| Instead of this | Do this |
|-----------------|---------|
| "Add animation" | "What does this communicate?" |
| "Add cards" | "What information needs this container?" |
| "Add gradients" | "What emotion needs this treatment?" |
| "Add more sections" | "Has the visitor learned what they need?" |

### The Decision Filter

Before adding any element, ask:

1. Does this **reduce friction**?
2. Does this **increase confidence**?
3. Does this **reward curiosity**?
4. Does this **guide attention**?
5. Does this **create momentum**?

If the answer to all five is no, delete it.

## 03 — Storytelling Framework

### Every page answers five questions

Without exception.

| Question | Answer | Section Type |
|----------|--------|--------------|
| **Why should I care?** | The problem or opportunity | Statement, Hero |
| **Why should I trust you?** | Evidence of competence | Work, Metrics, Testimonial |
| **Can you actually do it?** | Capabilities, proof | Capabilities, Case Study |
| **How does it work?** | Process, methodology | Process, Timeline |
| **What's next?** | Clear action | CTA |

### The Narrative Arc

Every page follows this flow:

```
Curiosity
  ↓
Understanding
  ↓
Confidence
  ↓
Trust
  ↓
Action
```

### The Page Test

If a page doesn't answer all five questions, it's incomplete.

If a page answers them out of order, it's confusing.

If a page answers them twice, it's redundant.

## 04 — Layout Rhythm

### How information flows

Information doesn't flow in equal steps.

It flows in **rhythm**.

### The Rhythm Principle

Every section has one of three densities:

| Density | Purpose | Example |
|---------|---------|---------|
| **Breath** | Let the eye rest | Statement, single sentence |
| **Content** | Deliver information | Project, capabilities |
| **Impact** | Create emphasis | Hero, metrics, CTA |

### The Rhythm Pattern

A page should alternate between these densities:

```
Impact (Hero)
  ↓
Breath (Statement)
  ↓
Content (Project)
  ↓
Breath (Single image)
  ↓
Content (Capabilities)
  ↓
Impact (Metrics)
  ↓
Breath (Quote)
  ↓
Impact (CTA)
```

### Never do this

```
Content (80px)
Content (80px)
Content (80px)
Content (80px)
Content (80px)
```

Robotic. Exhausting. Forgettable.

### Always do this

```
Impact (160px)
Breath (64px)
Content (240px)
Breath (80px)
Content (180px)
Impact (120px)
```

Variable. Musical. Memorable.

### Section Spacing

| Transition | Spacing |
|------------|---------|
| Impact → Breath | 64–80px |
| Breath → Content | 120–160px |
| Content → Breath | 80–120px |
| Breath → Impact | 120–160px |
| Impact → Impact | 64–80px (rare) |

## 05 — Typography

### Editorial rules

Typography carries 80% of the visual weight.

If the typography is beautiful, the page is beautiful.

If the typography is broken, no amount of images or colors will save it.

### Two Voices

| Voice | Font | Role |
|-------|------|------|
| **Editorial** | DM Serif Display | Headlines, statements, philosophy |
| **Technical** | Geist | Body, UI, data, navigation |

These two voices never compete.

Editorial speaks in large, confident statements.
Technical speaks in clear, precise information.

### The Type Scale

| Level | Size | Use |
|-------|------|-----|
| **Monumental** | `clamp(3rem, 8vw, 8rem)` | Hero headline |
| **Editorial** | `clamp(2rem, 5vw, 4.5rem)` | Section headline |
| **Statement** | `clamp(1.75rem, 4vw, 3.5rem)` | Editorial pause |
| **Subhead** | `clamp(1.25rem, 2.5vw, 2rem)` | Supporting text |
| **Body** | `1rem` | Default text |
| **Caption** | `0.875rem` | Labels, helpers |
| **Overline** | `0.75rem` | Category labels |

### The Rule of One

**One editorial headline per viewport.**

Not two. Not three. One.

If you need another headline, the visitor should scroll first.

### Line Height Rules

| Context | Line Height |
|---------|-------------|
| Headlines (Monumental) | 1.02 |
| Headlines (Editorial) | 1.08 |
| Statements | 1.15 |
| Body text | 1.6 |

Tight headlines. Loose body. Never the reverse.

### Line Length

| Context | Max Width |
|---------|-----------|
| Headlines | None (let them be big) |
| Body text | 65ch |
| Quotes | 55ch |

If a line of body text exceeds 65 characters, it becomes hard to read.

## 06 — White Space

### When to leave things empty

White space is not empty space.

It is **active composition**.

It tells the visitor: "This is important. Look here."

### The White Space Principle

The more white space around an element, the more important it is.

| Importance | White Space |
|------------|-------------|
| Normal | Standard section spacing |
| Important | 1.5× standard spacing |
| Critical | 2×+ standard spacing |
| Hero | Full viewport height |

### The Empty Page Test

Before adding anything to a page, ask:

> "Does this section earn the space it takes?"

If a section doesn't make the visitor feel something, learn something, or do something — delete it.

### The Apple Principle

Apple's homepage has more white space than content.

That's not an accident.

That's the design.

## 07 — Visual Tension

### Contrast creates beauty

Not gradients. Not decorations. **Contrast.**

### The Tension Pairs

Every viewport should contain at least one tension pair:

| Pair | Example |
|------|---------|
| **Large / Small** | Huge headline + tiny label |
| **Dark / Light** | Dark section + light section |
| **Empty / Dense** | Massive whitespace + concentrated info |
| **Near / Far** | Close-up detail + wide shot |
| **Fast / Slow** | Quick animation + long pause |
| **Sharp / Soft** | Crisp typography + blurred background |

### The Contrast Rule

If two elements feel the same weight, size, or density — change one of them.

Same weight = boring.
Different weight = interesting.

### The Monochrome Test

Remove all color from your page.

If the composition still works, the tension is correct.

If it looks flat, the tension is missing.

## 08 — Recognition System

### What makes Aesthetix unmistakable

Imagine I blur your website.

Could I still recognize it?

If not, the design has no identity.

### The Six Markers

Every Aesthetix page contains these recognition markers:

| Marker | Expression |
|--------|------------|
| **Huge serif headline** | DM Serif Display, massive, tight leading |
| **Tiny uppercase label** | Overline, 0.75rem, wide tracking |
| **Thin divider** | 1px, zinc-200, full width |
| **Massive whitespace** | More space than content |
| **One blue accent** | Brand violet, used sparingly |
| **Architectural composition** | Asymmetric, structural, editorial |

### The Blur Test

1. Take a screenshot of your page
2. Apply Gaussian blur (10px)
3. Ask: "Is this Aesthetix?"

If yes → the identity is strong.
If no → add more recognition markers.

### The Consistency Rule

These six markers appear on **every page**.

Not sometimes. Every time.

That's how recognition is built.

## 09 — Visual Hierarchy Score

### Every viewport needs a clear hierarchy

If everything scores 80, nothing stands out.

### The Score System

Every element in a viewport gets a score:

| Score | Element | Visual Weight |
|-------|---------|---------------|
| 100 | Headline | Largest, highest contrast |
| 80 | Image | Full attention, visual anchor |
| 70 | CTA | Action-oriented, distinct |
| 45 | Body | Supporting information |
| 20 | Metadata | Labels, categories, dates |

### The Hierarchy Test

In every viewport:
- One element scores 100
- One element scores 80
- One element scores 70
- Everything else scores below 50

If two elements both score 100, the hierarchy is broken.

### The Fix

If hierarchy is broken:
1. Make one element bigger
2. Make the other smaller
3. Or remove one entirely

Never have two focal points in one viewport.

## 10 — Editorial Rules

### Magazine designers know something web designers don't

Every spread has three elements:

```
Hero
Support
Silence
```

Not:

```
Hero
Hero
Hero
Hero
```

### The Editorial Spread

| Element | Role | Example |
|---------|------|---------|
| **Hero** | The main visual or statement | Headline, full-width image |
| **Support** | Supporting information | Body text, metadata, tags |
| **Silence** | Breathing room | White space, empty area |

### The Silence Principle

Silence is part of composition.

Without silence, there is no emphasis.

Without emphasis, there is no hierarchy.

Without hierarchy, there is no story.

### The Spread Test

Look at any viewport. Ask:

1. Where is the hero?
2. Where is the support?
3. Where is the silence?

If you can't answer all three, the spread is incomplete.

## 11 — Motion

### When things move. When they don't.

Motion communicates.

It doesn't decorate.

### The Motion Principle

If an animation doesn't communicate something, delete it.

| Motion | Communication |
|--------|---------------|
| Fade up | "This is arriving." |
| Scale | "This is important." |
| Stagger | "These are related." |
| Parallax | "This has depth." |

### The Motion Budget

Every page gets **three** types of motion:

1. **Entrance** — Elements fade up as they enter viewport
2. **Transition** — Page transitions (if any)
3. **Hover** — Interactive feedback

That's it.

No scrolling animations. No parallax on every section. No bouncing elements.

### Timing

| Duration | Use |
|----------|-----|
| 120ms | Hover effects |
| 200ms | Micro-interactions |
| 400ms | Entrance animations |
| 600ms | Page transitions |

### The Reduced Motion Rule

If `prefers-reduced-motion` is enabled, all non-essential animation is disabled.

Essential = entrance animations that help the visitor understand the page structure.
Non-essential = everything else.

## 12 — Composition

### The visual grammar of Aesthetix

Composition is how elements relate to each other in space.

It's the difference between a page that feels "right" and a page that feels "off."

### The Grid

Aesthetix uses a **12-column grid** on desktop, **4-column** on mobile.

But the grid is a guide, not a cage.

### Compositional Patterns

| Pattern | Use | Example |
|---------|-----|---------|
| **Full width** | Hero, statements | Headline spans entire viewport |
| **Offset left** | Capabilities | Content left, labels right |
| **Offset right** | Alternate | Labels left, content right |
| **Centered** | Statements, CTAs | Maximum emphasis |
| **Split** | Projects | Image + text side by side |

### Asymmetry

Symmetry is boring.

Asymmetry is interesting.

| Symmetric (avoid) | Asymmetric (prefer) |
|-------------------|---------------------|
| □ □ □ | ██████████████ |
| □ □ □ | ██ |
| □ □ □ | ██████ |

### The Focal Point

Every section has **one focal point**.

Not two. One.

The focal point gets:
- Most white space
- Largest type
- Highest contrast

Everything else is supporting.

### Breaking the Grid

You can break the grid when:
- Creating a "wow" moment
- Emphasizing a single element
- Creating visual surprise

You should NOT break the grid when:
- Showing multiple items
- Displaying data
- Presenting navigation

## 13 — Signature Moments

### What makes a page memorable

Every great studio has one unforgettable moment.

| Studio | Signature Moment |
|--------|------------------|
| Apple | Huge product render |
| Stripe | Massive illustration |
| Vercel | Giant typography |
| Linear | Smooth motion |
| Nothing | Transparent design |

### The Aesthetix Signature

Aesthetix owns **architectural composition**.

Not animation. Not illustration. Not color.

**Composition itself.**

### The Three Moments

Every homepage has three signature moments:

| Scroll Position | Moment | Feeling |
|-----------------|--------|---------|
| **25%** | One impossible composition | "Wait, what?" |
| **60%** | One gigantic quote or metric | "Wow." |
| **85%** | One dramatic CTA | "I need to contact them." |

### The Memorability Test

24 hours after visiting, what will the visitor remember?

If the answer is "nice animations" → the design failed.
If the answer is "they really understand digital products" → the design succeeded.

## 14 — Homepage Blueprint

### The emotional arc

The homepage is not a list of sections.

It's a sequence of emotions.

| Emotion | Section | Composition |
|---------|---------|-------------|
| **Arrival** | Hero | Full viewport. Huge headline. Almost empty. |
| **Belief** | Statement | Single sentence. Massive whitespace. |
| **Evidence** | Featured Work | One project. Full-width image. Minimal text. |
| **Capability** | What We Build | Asymmetric editorial grid. No cards. |
| **Proof** | Metrics | 4 numbers. Dark background. Nothing else. |
| **Invitation** | CTA | One sentence. One button. Dark background. |

### The Emotion Flow

```
Arrival
  ↓
Belief
  ↓
Evidence
  ↓
Capability
  ↓
Proof
  ↓
Invitation
```

Each emotion builds on the last.

Arrival creates curiosity.
Belief creates understanding.
Evidence creates trust.
Capability creates confidence.
Proof creates desire.
Invitation creates action.

### The Viewport Budget

| Section | Viewports | Emotion |
|---------|-----------|---------|
| Hero | 1.0 | Arrival |
| Statement | 0.8 | Belief |
| Featured Work | 1.2 | Evidence |
| Capabilities | 1.5 | Capability |
| Metrics | 0.8 | Proof |
| CTA | 1.0 | Invitation |
| **Total** | **~6.3** | |

A visitor should scan the entire homepage in **6–7 scrolls**.

### What's NOT on the homepage

- Services (they're in the nav)
- Technology stack (proven through work)
- Team (they're in About)
- FAQ (they're in a dedicated page)
- Pricing (they're in Pricing)
- Client logos (they're in the work)
- Process (it's on the service pages)

The homepage is not a sitemap. It's a story.

## 15 — Visual Anchor Rule

### Every viewport has one visual anchor

The anchor is what the eye goes to first.

It can be:

| Anchor Type | Use |
|-------------|-----|
| **Typography** | Headlines, statements |
| **Image** | Product renders, screenshots |
| **Video** | Demos, walkthroughs |
| **Metric** | Numbers, data |
| **Quote** | Testimonials, philosophy |

### The One-Anchor Rule

Never more than one anchor per viewport.

If you have a huge headline AND a full-width image, one of them must shrink.

If you have a metric AND a quote, remove one.

### The Anchor Test

In every viewport:
1. What is the first thing the eye sees?
2. Is there only one answer?

If there are two answers, the anchor is broken.

### The Flexibility Principle

This rule gives freedom.

Sometimes the anchor is typography.
Sometimes it's an image.
Sometimes it's a metric.

The type changes. The principle doesn't.
