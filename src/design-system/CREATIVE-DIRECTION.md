## Design Manifesto

We don't design websites.

We design digital products that solve business problems through strategy, engineering, and thoughtful design.

Every interface should reduce complexity.
Every interaction should communicate intention.
Every detail should earn its place.
Every decision should reinforce trust.

We believe restraint is more powerful than decoration.
We believe systems scale better than trends.
We believe great products feel inevitable.

This is the philosophy behind Aesthetix Design Language.

## 01 — Vision

Aesthetix Studio exists to prove that digital products can be both technically excellent and emotionally resonant.

We are not a web agency.
We are not a design studio.

Our work sits at the intersection of:
- **Engineering discipline** — code that performs, scales, and endures
- **Design systems** — components that compose, not decorations that accumulate
- **Strategic thinking** — every pixel serves a business objective

The website is the first proof that this philosophy works.

## 04 — Visual Identity

### The Aesthetic: Editorial Product Architecture

Not minimalism. Not maximalism. Not brutalism.

**Editorial Product Architecture** combines:

- **Swiss editorial grids** — precision, hierarchy, rhythm
- **Museum-like whitespace** — breathing room as a design element
- **Product precision** — every element placed with intention
- **Engineering discipline** — systematic, not decorative
- **Architectural composition** — structure as beauty

|--------|----------------|

## 05 — Layout Philosophy

### Core Principle: Horizontal Before Vertical

Professional agencies define rhythm before sections.

We define:
- **Spacing rhythm** — consistent vertical cadence
- **Visual rhythm** — alternating density and breathing room
- **Narrative rhythm** — problem → insight → proof → action
- **Motion rhythm** — entrance timing, scroll choreography

### The Layout Equation

```
Every page = Section → Container → Stack → Content
```

### Section

The outermost wrapper. Controls:
- Vertical spacing (padding-top, padding-bottom)
- Background color or treatment
- Full-width or constrained
- Overflow behavior (hidden, visible, scroll)

### Container

Horizontal constraint. Controls:
- Max-width (standard: 1200px, wide: 1440px, narrow: 680px)
- Horizontal padding
- Content alignment

### Stack

Vertical gap management. Controls:
- Gap between children (4pt grid multiples)
- Optional dividers between items

### Cluster

Horizontal wrapping group. Controls:
- Gap between items
- Wrap behavior
- Alignment

### Split

Asymmetric left-right layout. Controls:
- Left content width
- Right content width
- Gap between
- Vertical alignment

### Grid

Column-based layout. Controls:
- Column count (2, 3, 4, 6, 12)
- Column gap
- Row gap

## 06 — Composition System

### Whitespace Ratios

| Context | Ratio |
|---------|-------|
| Section to section | 80–120px vertical |
| Container to edge | 20–32px horizontal |
| Headline to body | 16–24px |
| Body to body | 8–12px |
| Element to label | 4–8px |

### Density Levels

| Level | Description | Use Case |
|-------|-------------|----------|
| **Dense** | Minimal whitespace, tight spacing | Data tables, metrics |
| **Standard** | Normal spacing, comfortable reading | Most content |
| **Spacious** | Generous whitespace, breathing room | Hero, statements |
| **Monumental** | Maximum whitespace, editorial | Manifesto, philosophy |

### Compositional Balance

Every section should feel **intentionally asymmetric**.

Not centered-everything. Not justified blocks.

Instead:
- Headlines slightly left of center
- Body text constrained to reading width
- Images breaking the grid intentionally
- Numbers large and confident
- White space as a compositional element

## 07 — Typography as Voice

### The Two Voices

**Display (DM Serif Display):**
The editorial voice. Used for:
- Hero headlines
- Section statements
- Philosophical declarations
- Chapter titles

Character: Confident, refined, editorial.

**Interface (Geist):**
The technical voice. Used for:
- Body text
- UI labels
- Navigation
- Data
- Documentation

Character: Clear, precise, modern.

### Typographic Hierarchy

| Level | Font | Size | Weight | Use |
|-------|------|------|--------|-----|
| Display Hero | DM Serif | 64–80px | 400 | Hero statements |
| Display Lg | DM Serif | 48–56px | 400 | Section headlines |
| Display Md | DM Serif | 36–44px | 400 | Subsection headlines |
| Title Xl | Geist | 32–40px | 700 | Page titles |
| Title Lg | Geist | 28–32px | 700 | Section titles |
| Title Md | Geist | 24–28px | 600 | Card titles |
| Body Lg | Geist | 18–20px | 400 | Lead paragraphs |
| Body Md | Geist | 16px | 400 | Default body |
| Body Sm | Geist | 14px | 400 | Secondary text |
| Caption | Geist | 12px | 400 | Labels, helpers |
| Overline | Geist | 11–12px | 600 | Category labels |

### Typographic Rules

1. **Never center body text.** Left-align or right-align.
2. **Headlines can be center-aligned** only in editorial statements.
3. **Maximum line length:** 680px for body, 480px for quotes.
4. **Letter spacing:** Tighter for large type, normal for body.
5. **Line height:** Tighter for headlines (1.1–1.2), looser for body (1.6–1.75).

## 08 — Material & Light

### Surface Language

| Surface | Light Mode | Dark Mode |
|---------|------------|-----------|
| Background | `#FAFAF9` (warm white) | `#0D0D0C` (near black) |
| Card | `#FFFFFF` | `#161614` |
| Elevated | `#FFFFFF` + shadow | `#1C1C1A` + shadow |
| Sunken | `#F3F3F1` | `#1C1C1A` |

### Shadow Philosophy

Shadows are **depth cues**, not decorations.

| Level | Use |
|-------|-----|
| `shadow-xs` | Subtle lift (badges, small elements) |
| `shadow-sm` | Cards at rest |
| `shadow-md` | Cards on hover, dropdowns |
| `shadow-lg` | Modals, popovers |
| `shadow-xl` | Overlays, notifications |

### Light Direction

All shadows assume light comes from **top-center**.

This creates:
- Natural depth hierarchy
- Consistent visual weight
- Professional, product-like feel

### Blur & Glass

Use sparingly. Only for:
- Navigation backdrop (optional)
- Modal overlays
- Floating elements

Never for:
- Card backgrounds
- Section backgrounds
- Decorative effects

## 09 — Color Psychology

### The Palette Has Roles, Not Just Colors

| Role | Color | Emotion |
|------|-------|---------|
| **Brand (Violet)** | `#6150F6` | Innovation, intelligence, momentum |
| **Craft (Gold)** | `#C9A21F` | Premium, quality, craftsmanship |
| **Neutral** | Grays | Structure, stability, calm |
| **Success (Green)** | `#16A34A` | Confidence, completion |
| **Warning (Amber)** | `#D97706` | Attention, caution |
| **Error (Red)** | `#DC2626` | Urgency, action needed |
| **Info (Blue)** | `#2563EB` | Knowledge, assistance |

### Color Usage Rules

1. **Brand violet** is for interaction, not decoration.
2. **Craft gold** is for premium accents only (star ratings, special highlights).
3. **Neutrals** dominate 90% of the interface.
4. **Status colors** are functional, not decorative.
5. **Never use color to convey meaning alone** — always pair with icon or text.

### Dark Mode Philosophy

Dark mode is not inverted light mode.

In dark mode:
- Backgrounds become deeper, not just darker
- Shadows become more subtle
- Brand colors become lighter (more luminous)
- Text becomes softer (not pure white)

## 11 — Motion Philosophy

### Core Principle: Motion Communicates, Not Decorates

Every animation must answer: **What does this communicate?**

| Motion Type | Communication |
|-------------|---------------|
| **Entrance** | "This is arriving." |
| **Exit** | "This is leaving." |
| **Transition** | "We're moving between states." |
| **Feedback** | "Your action was received." |
| **Focus** | "Look here." |
| **Progress** | "Something is happening." |

### Timing

| Duration | Use |
|----------|-----|
| 120ms | Micro-interactions (hover, focus) |
| 180ms | Standard transitions |
| 280ms | Complex state changes |
| 400ms | Page-level transitions |

### Easing

| Curve | Use |
|-------|-----|
| `ease-standard` (0.4, 0, 0.2, 1) | Default, natural feel |
| `ease-entrance` (1, 1, 0.3, 1) | Elements arriving |
| `ease-exit` (0.7, 0, 0.84, 0) | Elements leaving |

### Motion Rules

1. **Never animate for decoration.** Every animation must communicate.
2. **Stagger related elements** (50–80ms between items).
3. **Use `whileInView`** for scroll-triggered animations.
4. **Use `viewport={{ once: true }}`** — don't re-animate on scroll back.
5. **Respect `prefers-reduced-motion`** — disable non-essential animation.
6. **Keep hover effects subtle** — scale(0.98), shadow increase, color shift.
7. **Page transitions** should be 280–400ms with entrance easing.

### Scroll Choreography

For complex sections, stagger elements:

```
Headline enters (0ms delay)
  ↓
Subline enters (100ms delay)
  ↓
CTA enters (200ms delay)
  ↓
Supporting content enters (300ms delay)
```

## 12 — Editorial System

### Copy Tone

| Quality | Expression |
|---------|------------|
| **Confident** | We state things directly. No hedging. |
| **Precise** | We use specific numbers, not vague claims. |
| **Technical** | We don't dumb things down. |
| **Brief** | We say more with fewer words. |
| **Active** | We use active voice, not passive. |

### Headline Philosophy

Headlines should be **declarative statements**, not questions.

**Good:**
- "Your website should be your best salesperson."
- "Design is a business decision."
- "We build products that scale."

**Bad:**
- "Ready to transform your digital presence?"
- "Looking for a creative partner?"
- "Let's build something amazing together."

### Body Copy Rules

1. **Maximum paragraph length:** 3 sentences.
2. **One idea per paragraph.**
3. **Lead with the benefit,** not the feature.
4. **Use specific numbers:** "42% conversion lift" not "significant improvement."
5. **No jargon without explanation.**
6. **No superlatives without proof.**

### Content Hierarchy

Every section should follow:

```
Overline (category label)
  ↓
Headline (main statement)
  ↓
Subline (supporting detail)
  ↓
Body (explanation)
  ↓
CTA (action)
```

## 14 — Interaction Philosophy

### Micro-Interactions

Every interactive element should provide **immediate, meaningful feedback**.

| State | Feedback |
|-------|----------|
| **Rest** | Clear, visible, inviting |
| **Hover** | Subtle lift, color shift, shadow increase |
| **Active/Pressed** | Scale down slightly (0.98), darker shade |
| **Focus** | Ring visible, border highlight |
| **Disabled** | Reduced opacity (0.4), cursor not-allowed |
| **Loading** | Spinner replaces content, button disabled |

### Button Behavior

| Variant | Hover | Active |
|---------|-------|--------|
| Brand | Darker shade + shadow increase | Scale 0.98 |
| Primary | Slight darken | Scale 0.98 |
| Ghost | Background appear | Scale 0.98 |
| Link | Underline appear | — |

### Form Behavior

| Element | Feedback |
|---------|----------|
| **Input focus** | Ring appears, border color shifts |
| **Input error** | Red border, error message below |
| **Input success** | Green checkmark (if applicable) |
| **Select open** | Dropdown animates in (fade + scale) |
| **Submit** | Button shows loading state |

## 15 — Experience Principles

### The Non-Negotiables

1. **Performance first.** Every page must load in under 2 seconds.
2. **Accessibility always.** WCAG AA compliance is not optional.
3. **Mobile-native.** Design for mobile, enhance for desktop.
4. **Content-first.** Typography and content drive layout, not the reverse.
5. **Systematic.** Every element must be explainable as part of the system.

### Quality Standards

| Metric | Target |
|--------|--------|
| Page load | < 2s |
| First Contentful Paint | < 1s |
| Largest Contentful Paint | < 2.5s |
| Cumulative Layout Shift | < 0.1 |
| Time to Interactive | < 3s |
| Lighthouse score | > 95 |
| WCAG compliance | AA |

### The 5-Second Test

Within 5 seconds of landing on the homepage, a visitor should be able to answer:
1. **What does this company do?**
2. **Who is it for?**
3. **Why should I care?**

If they can't, the hero has failed.

## 17 — Anti-Patterns

### What Aesthetix is NOT

| Anti-Pattern | Why We Reject It |
|--------------|------------------|
| **Card overload** | Every section shouldn't be cards. Vary layouts. |
| **Gradient blobs** | Decorative gradients without purpose are noise. |
| **Centered everything** | Centered text everywhere creates visual monotony. |
| **Stock photography** | Generic images destroy credibility. |
| **Animation for show** | Motion without communication is distraction. |
| **Feature lists** | Bullet lists don't tell stories. |
| **Jargon stacking** | "AI-powered synergistic solutions" means nothing. |
| **Social proof overload** | 20 testimonials don't convince better than 3. |
| **Pricing tables everywhere** | Not every page needs pricing. |
| **Chatbot popups** | Intrusive, not helpful. |
| **Cookie banners (ugly)** | Compliance doesn't mean ugly. |
| **Loading screens** | The site should be fast enough to not need them. |

### The Simplicity Test

Before adding any element, ask:
1. **Does this communicate something?**
2. **Does this help the visitor?**
3. **Does this earn its space?**

If the answer to all three is no, remove it.

## 18 — Creative Checklist

### Pre-Launch Validation

Use this checklist before any page goes live:

#### Visual
- [ ] Whitespace ratio is consistent (80–120px between sections)
- [ ] Typography hierarchy is clear (3 levels maximum per section)
- [ ] No more than 2 font sizes per section
- [ ] Images are high-res, properly cropped, on-brand
- [ ] No stock photography
- [ ] Color usage follows role system
- [ ] Shadows are consistent (light from top-center)

#### Layout
- [ ] Section → Container → Stack → Content pattern followed
- [ ] Mobile layout tested (no horizontal scroll)
- [ ] No card-only layouts (varied composition)
- [ ] Editorial pauses exist (statement sections)
- [ ] Grid is consistent (4pt base)

#### Motion
- [ ] All animations have communication purpose
- [ ] Stagger timing is consistent (50–80ms)
- [ ] `prefers-reduced-motion` respected
- [ ] No animation on page load (except hero entrance)
- [ ] Hover effects are subtle (scale 0.98, shadow)

#### Content
- [ ] Headlines are declarative, not questions
- [ ] Body copy is < 3 sentences per paragraph
- [ ] Specific numbers used, not vague claims
- [ ] One idea per section
- [ ] CTA is clear and actionable

#### Technical
- [ ] Page loads in < 2 seconds
- [ ] Lighthouse score > 95
- [ ] WCAG AA compliance
- [ ] No console errors
- [ ] Responsive across all breakpoints

#### Narrative
- [ ] 5-second test passes (what, who, why)
- [ ] Story arc complete (problem → proof → action)
- [ ] Emotional progression clear (curiosity → trust → action)
- [ ] No section exists without purpose

## Appendix: Aesthetix Page Template

Every Aesthetix page should follow this general structure:

```
1. Navigation
   - Minimal, fixed, clear hierarchy

2. Hero
   - Editorial headline (DM Serif Display)
   - Supporting subline (Geist)
   - Primary CTA
   - Trust signal (optional)

3. Problem / Insight
   - Statement primitive
   - "Why this matters" editorial section

4. Philosophy / Approach
   - How we think, not just what we do
   - Editorial layout with imagery

5. Capabilities
   - Feature grid or split layouts
   - Not just a list of services

6. Selected Work
   - Case study primitives
   - Proof of capability

7. Metrics / Results
   - Numbers that demonstrate impact
   - Metric primitive

8. Process
   - Timeline primitive
   - How we work

9. Testimonials
   - Quote primitives
   - Social proof

10. CTA
    - Clear, compelling action
    - Final emotional beat
```
